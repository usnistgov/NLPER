The NIST Randomness Beacon

Source: https://csrc.nist.gov/Projects/Interoperable-Randomness-Beacons

A randomness beacon produces timed outputs of fresh public randomness. This randomness can be used to promote public auditability of public processes that depend on random selections (audits, lotteries, etc).

The first public prototype of the NIST randomness beacon was launched in 2013. In 2019, the Interoperable Randomness Beacons project at NIST published a new reference for randomness beacons (version 2.0).

The NIST Randomness Beacon creates a new pulse every minute, with an “output value” with 512 random bits. Each pulse is hash-chained to previous pulses, and contains other metadata to support several security features. Also, every past pulse must be available for public scrutiny.

For internationl interoperability, each pulse has a timestamp in Universal Time Coordinate (UTC) format.
